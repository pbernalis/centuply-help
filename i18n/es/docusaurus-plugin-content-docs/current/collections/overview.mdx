# Collections: grupos de contenido reutilizable y listas dinámicas

Collections te ayudan a organizar contenido repetido en tu sitio web.

Si tienes muchos elementos similares, como servicios, productos, artículos, ofertas, miembros del equipo o proyectos, puedes gestionarlos de forma más clara mediante Collections.

## Cuándo usar Collections

Usa Collections cuando quieras organizar:

- servicios,
- productos,
- proyectos,
- portfolio items,
- miembros del equipo,
- testimonials,
- ofertas,
- contenido tipo blog,
- ubicaciones,
- FAQs,
- bloques reutilizables.

Si el mismo tipo de contenido se repite muchas veces, probablemente encaja en una Collection.

## Qué es una Collection

Una Collection es un grupo de items similares.

Ejemplo de Collection: **Services**

Items dentro de la Collection:

- Website Design,
- SEO Setup,
- Logo Design,
- Booking Setup,
- Support Package.

Cada item tiene sus propios fields, como título, descripción, imagen, precio o enlace.

## Por qué es útil

Collections ayudan porque:

- mantienen el contenido organizado,
- reducen cambios duplicados,
- facilitan la gestión de muchos items,
- permiten dynamic lists,
- crean una estructura más limpia,
- hacen que el sitio sea más fácil de actualizar.

Si cambias un item en la Collection, puede actualizarse donde se use.

## Paso 1 — Abre Collections

Abre la aplicación de Centuply, selecciona tu sitio y abre **Collections**.

Allí verás collections existentes o una opción para crear una nueva.

## Paso 2 — Crea una nueva Collection

Haz clic en **Create collection** o **Add collection**.

Usa un nombre claro, como:

- Services,
- Products,
- Team,
- Testimonials,
- Offers,
- Locations,
- Portfolio,
- FAQs.

El nombre debe explicar qué tipo de contenido contiene.

## Paso 3 — Añade fields

Los fields son los datos que tiene cada item.

Ejemplo para Services:

- title,
- short description,
- full description,
- image,
- price,
- CTA label,
- CTA link.

Ejemplo para Team:

- name,
- role,
- bio,
- photo,
- social link.

Mantén los fields útiles y no excesivos.

## Paso 4 — Añade items

Después añade items dentro de la Collection.

Por ejemplo, en una Services Collection puedes añadir:

- Haircut,
- Hair color,
- Bridal styling,
- Hair treatment.

Para cada item, completa los fields con cuidado.

## Paso 5 — Conecta Collection con una sección

Una Collection puede usarse en una sección del sitio.

Ejemplos:

- Services grid,
- Portfolio gallery,
- Team section,
- Testimonials carousel,
- Offers list,
- Locations list.

Así la sección muestra automáticamente los items de la Collection.

## Paso 6 — Elige layout de visualización

Según la sección, puedes elegir distintos layouts.

Ejemplos:

- grid,
- cards,
- list,
- carousel,
- masonry,
- compact list,
- featured item,
- comparison layout.

Elige un layout que encaje con el contenido y la marca.

## Paso 7 — Revisa la dynamic list

Si la sección es dinámica, toma contenido de la Collection.

Revisa:

- si aparecen todos los items,
- si el orden es correcto,
- si los enlaces funcionan,
- si las imágenes cargan,
- si los CTAs son correctos,
- si no aparecen fields vacíos.

## Paso 8 — Organiza el orden de items

El orden de items afecta la presentación.

Pon primero los más importantes.

Ejemplo para servicios:

1. servicio principal,
2. servicio más rentable,
3. servicio premium,
4. servicios complementarios.

El orden debe ayudar al visitante a entender rápidamente lo que ofreces.

## Paso 9 — Revisa apariencia móvil

Collections suelen aparecer como grids o cards, por eso la revisión móvil es importante.

Comprueba:

- si las cards son claras,
- si los títulos no se cortan,
- si las imágenes se ven bien,
- si los CTAs son fáciles de tocar,
- si la lista no se vuelve demasiado larga.

## Paso 10 — Actualiza items cuando cambie el contenido

Cuando cambie un servicio, producto u oferta, actualiza el item correspondiente en la Collection.

No cambies solo el texto en una sección si el mismo item aparece en otros lugares.

Así mantienes el contenido consistente.

## Buenas prácticas

- Usa Collections para contenido repetido.
- Mantén nombres claros.
- No añadas fields innecesarios.
- Revisa enlaces y CTAs.
- Mantén un orden intencional de items.
- Revisa apariencia móvil.
- Actualiza la Collection en lugar de cambiar manualmente muchas secciones.
- No muestres demasiados items en la homepage.

## Preguntas frecuentes

### ¿Cuándo necesito una Collection?

Cuando tienes muchos items similares, como servicios, productos, testimonials o proyectos.

### ¿Puedo usar la misma Collection en varias páginas?

Sí, si está soportado. Esto ayuda a mantener el contenido consistente.

### ¿Qué es una dynamic list?

Es una sección que muestra automáticamente items de una Collection.

### ¿Todos los sitios necesitan Collections?

No. Son útiles sobre todo cuando hay contenido repetido o estructurado.

### ¿Qué hago si un item no aparece correctamente?

Revisa si los fields están completos, si el item está activo y si la sección está conectada con la Collection correcta.
