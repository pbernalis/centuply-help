# Site Cockpit: gestionar tu sitio desde un solo lugar

Site Cockpit es el área central de control para tu sitio web en Centuply.

Desde aquí puedes ver el estado del sitio, abrir el editor, revisar publicación, dominio, SEO, formularios, leads y ajustes principales.

## Cuándo usar Site Cockpit

Usa Site Cockpit cuando quieras:

- abrir tu sitio rápidamente,
- continuar editando,
- ver si el sitio está draft o published,
- revisar advertencias básicas,
- abrir Visual Editor,
- ir a Pages, SEO, Forms o Publish,
- ver rápidamente qué necesita atención antes del live.

## Qué ves en Site Cockpit

Según el estado del sitio, puedes ver:

- nombre del sitio,
- preview URL,
- publish status,
- domain status,
- acciones principales,
- botón Edit,
- botón Publish,
- site health checks,
- actividad reciente,
- resumen de forms o leads,
- recordatorios de SEO o setup.

El objetivo es darte una visión rápida sin buscar en muchos menús.

## Paso 1 — Abre Site Cockpit

Abre la aplicación de Centuply y selecciona el sitio que quieres gestionar.

Site Cockpit aparece como página principal del proyecto o página overview.

Si tienes varios sitios, asegúrate de abrir el proyecto correcto.

## Paso 2 — Revisa el estado del sitio

El estado muestra en qué fase está el sitio.

Ejemplos:

- Draft,
- Generating,
- Ready to edit,
- Published,
- Needs attention,
- Payment pending,
- Domain pending.

Si el sitio no está listo para publicar, Cockpit puede mostrar qué falta.

## Paso 3 — Abre Visual Editor

Desde Site Cockpit puedes abrir Visual Editor.

Úsalo cuando quieras cambiar:

- textos,
- imágenes,
- secciones,
- botones,
- páginas,
- apariencia,
- contenido principal.

Si el sitio aún está en draft, puedes seguir editando antes de publicar.

## Paso 4 — Revisa Pages y estructura

Site Cockpit puede darte acceso rápido a las páginas del sitio.

Revisa:

- si existen todas las páginas importantes,
- si falta Contact page,
- si los servicios tienen la página correcta,
- si los enlaces del menú funcionan,
- si hay páginas vacías o sin terminar.

Una estructura clara ayuda a que el sitio se vea más profesional.

## Paso 5 — Revisa recordatorios SEO

Si hay recordatorios SEO, revisa qué necesita mejora.

Normalmente debes revisar:

- page title,
- meta description,
- headings,
- datos de local SEO,
- imágenes,
- keywords básicas,
- social preview.

No todo tiene que estar perfecto desde el primer día, pero lo básico debe estar correcto.

## Paso 6 — Revisa formularios y leads

Si el sitio tiene formularios, Site Cockpit puede guiarte a las áreas relevantes.

Antes de publicar, revisa:

- si los formularios aparecen correctamente,
- si envían email,
- si el email destinatario es correcto,
- si hay mensaje de confirmación,
- si los envíos de prueba quedan registrados.

## Paso 7 — Revisa publish readiness

Antes de hacer publish, revisa si el sitio está listo.

Checklist útil:

- textos correctos,
- imágenes correctas,
- vista móvil limpia,
- botones correctos,
- formularios funcionando,
- email correcto,
- datos SEO básicos,
- dominio correcto o URL temporal.

Site Cockpit ayuda a ver rápidamente qué queda pendiente.

## Paso 8 — Revisa dominio

Si tienes dominio personalizado, revisa el estado de conexión.

Estados posibles:

- Not connected,
- Pending,
- Verified,
- SSL pending,
- Live.

Si el dominio no está listo, abre Domain o Publish settings y revisa las instrucciones DNS.

## Paso 9 — Revisa actividad reciente

Si existe sección de actividad o cambios recientes, úsala para recordar qué cambió recientemente.

Puede ayudarte a ver:

- cuándo se publicó el sitio,
- qué cambió por última vez,
- si el sitio fue actualizado,
- si alguna acción necesita seguimiento.

## Paso 10 — Continúa desde el lugar correcto

Site Cockpit es útil porque evita que te pierdas.

Si ves una advertencia SEO, abre SEO.

Si ves una advertencia de dominio, abre Domain.

Si el sitio está draft, abre Editor.

Si todo está listo, ve a Publish.

## Buenas prácticas

- Revisa Site Cockpit antes de cada publish.
- No dependas solo del desktop preview.
- Abre mobile preview con frecuencia.
- Revisa advertencias antes de ignorarlas.
- Mantén claro el estado del sitio.
- Revisa formularios después de cambios.
- Usa Cockpit como centro diario de control del sitio.

## Preguntas frecuentes

### ¿Site Cockpit es lo mismo que Editor?

No. Cockpit es el overview central. Editor es donde cambias contenido y apariencia.

### ¿Debo revisarlo antes de publicar?

Sí. Conviene revisar estado, advertencias, formularios, SEO y dominio antes de publicar.

### ¿Qué significa draft site?

Significa que el sitio aún no está publicado o que estás trabajando en cambios antes de hacerlos live.

### Si todo se ve correcto en Cockpit, ¿puedo publicar?

Casi. Haz también una revisión manual en desktop y móvil antes de publicar.

### ¿Puedo gestionar varios sitios?

Sí, si tu cuenta tiene varios proyectos. Solo asegúrate de seleccionar el sitio correcto antes de hacer cambios.
