# Storyboard View: recomendaciones de estructura con IA

Storyboard View te ayuda a ver la estructura general de tu sitio web como un mapa.

En lugar de mirar solo una página cada vez, puedes ver qué páginas existen, cómo encajan entre sí y qué páginas podrían faltar.

## Cuándo usar Storyboard View

Usa Storyboard View cuando quieras:

- ver la estructura completa del sitio,
- entender si falta una página importante,
- organizar mejor el contenido,
- añadir páginas recomendadas,
- mejorar el recorrido del visitante,
- comprobar si el sitio está listo para publicar.

## Qué muestra

Storyboard View puede mostrar:

- las páginas principales del sitio,
- la posición de cada página en la estructura,
- páginas recomendadas que pueden faltar,
- si existe cobertura básica para Home, Services, Contact o páginas legales,
- cómo puede moverse un visitante dentro del sitio.

## Paso 1 — Abre Pages Manager

Abre la aplicación de Centuply, selecciona tu sitio y ve a **Pages** o **Pages Manager**.

Desde allí, abre Storyboard View si está disponible.

## Paso 2 — Revisa la estructura del sitio

En Storyboard View, observa qué páginas existen.

Comprueba si el sitio tiene al menos:

- Home,
- Services o una página equivalente,
- Contact,
- páginas de confianza o información,
- páginas legales si hacen falta.

## Paso 3 — Revisa las recomendaciones de IA

Centuply puede mostrar sugerencias de páginas que faltan.

Ejemplos:

- FAQ,
- Pricing,
- Gallery,
- Booking,
- Blog,
- Privacy Policy,
- Terms,
- Locations,
- páginas de detalle de servicios.

No necesitas añadir todas las recomendaciones. Elige solo las páginas que tengan sentido para el negocio.

## Paso 4 — Añade una página recomendada

Si una página recomendada es útil, añádela desde Storyboard View o Pages Manager.

Después de añadirla, abre la página en el editor y revisa:

- título,
- secciones,
- textos,
- imágenes,
- CTA,
- detalles SEO.

## Paso 5 — Revisa el recorrido del visitante

Piensa en cómo se mueve un visitante por el sitio.

Un recorrido simple es:

1. Ve la página de inicio.
2. Entiende qué ofreces.
3. Ve servicios o productos.
4. Encuentra señales de confianza.
5. Encuentra datos de contacto.
6. Reserva, contacta o compra.

Si falta algún paso, quizá necesitas una nueva página o una sección mejor.

## Paso 6 — No sobrecargues el sitio

Las recomendaciones ayudan, pero no son obligatorias.

Un sitio pequeño de negocio no siempre necesita muchas páginas.

Es mejor tener pocas páginas claras que muchas páginas con contenido débil o insuficiente.

## Buenas prácticas

- Mantén una estructura simple.
- Añade páginas que ayuden al visitante.
- No añadas páginas solo porque se sugieren.
- Comprueba que cada página tenga un propósito claro.
- Abre cada nueva página en el editor antes de publicar.
- Revisa la navegación móvil.
- Añade páginas legales cuando hagan falta.

## Preguntas frecuentes

### ¿Debo añadir todas las páginas recomendadas?

No. Añade solo las páginas que realmente ayudan al sitio y al visitante.

### ¿Qué pasa si una página recomendada no encaja?

Puedes simplemente ignorarla.

### ¿Storyboard View es lo mismo que Pages Manager?

No exactamente. Pages Manager gestiona páginas, mientras que Storyboard View ayuda a ver la estructura y las recomendaciones de forma más visual.

### ¿Cuándo necesito una página FAQ?

Cuando los clientes hacen a menudo las mismas preguntas o cuando quieres reducir dudas antes del contacto o la reserva.

### ¿Cuándo necesito páginas legales?

Cuando el sitio recoge datos, usa formularios, cookies, pagos o necesita política de privacidad y términos.
