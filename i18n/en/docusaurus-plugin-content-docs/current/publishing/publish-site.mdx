# Publishing: publish your website

Publishing is the step that makes your website available online for visitors.

Before publishing, it is important to check that pages, text, images, contact details, SEO, and mobile view are correct.

## When to use Publish

Use Publish when:

- you have completed the main edits,
- you have reviewed the website in preview,
- contact details are correct,
- the main pages are ready,
- you have checked desktop and mobile,
- you want the website to go live,
- you want to publish changes to an already live website.

## Draft and live website

Before publishing, the website may be in draft mode.

The draft is the version you see and edit inside the editor.

The live website is the version visitors see online.

If you make changes to a website that is already live, you may need to publish again for those changes to appear online.

## Step 1 — Open your website

Open the Centuply app and select the website you want to publish.

Open the Site Cockpit or the Visual Editor.

## Step 2 — Check the main pages

Before clicking Publish, check the main pages:

- Home,
- Services,
- About,
- Contact,
- Booking, if available,
- Blog, if available,
- Legal pages, if needed.

Each page should have a clear purpose and should not be empty or unfinished.

## Step 3 — Check text and contact details

Read the website like a visitor.

Check:

- business name,
- phone number,
- email,
- address,
- opening hours,
- prices,
- services,
- buttons,
- forms,
- links.

Contact details must be completely accurate.

## Step 4 — Check images and brand

Make sure the images and brand look professional.

Check:

- logo,
- colors,
- fonts,
- hero image,
- gallery,
- service images,
- contrast,
- overall style.

If something looks unfinished, fix it before publishing.

## Step 5 — Check mobile preview

Many visitors will open the website from a phone.

Before publishing, check:

- whether headings fit,
- whether buttons are easy to tap,
- whether images crop correctly,
- whether the menu works,
- whether forms are easy to read,
- whether the page does not feel overloaded.

## Step 6 — Check SEO

Open the SEO section and check that the main pages have:

- SEO title,
- meta description,
- correct location, if it is a local business,
- clear content,
- correct page names.

Everything does not need to be perfect on day one, but the main pages should be correct.

## Step 7 — Check forms and booking

If the website has forms or booking, test them before publishing.

Check:

- whether the form opens,
- whether the fields are correct,
- whether the recipient email is correct,
- whether the confirmation message is clear,
- whether the booking flow works,
- whether CTAs go to the right place.

## Step 8 — Click Publish

When you are ready, click **Publish**.

Centuply will start the publishing process.

Depending on the website setup, it may take a few minutes for changes to appear online.

## Step 9 — Check the live website

After publishing, open the live URL and check again:

- homepage,
- main pages,
- menu,
- buttons,
- forms,
- mobile view,
- images,
- links.

Do not rely only on preview. Check the real live version too.

## Publish changes

If the website is already published and you make new edits, you may see an option such as **Publish changes**.

This means you are not publishing a new website from the beginning. You are updating the already live website with the latest changes.

## Best practices

- Do not publish without a final review.
- Always check mobile.
- Test forms and links.
- Do not leave empty pages.
- Check SEO title and description.
- Make sure the live website opens correctly.
- Keep the first updates simple and stable.

## Frequently asked questions

### Can I edit the website after publishing?

Yes. You can make changes and publish those changes again.

### Do changes appear immediately?

Usually they appear quickly, but sometimes it may take a few minutes.

### Do I need a custom domain before publishing?

Not necessarily. You can publish first and connect a custom domain later.

### What should I check before publishing?

Pages, text, images, contact details, SEO, mobile view, forms, and links.

### What if something looks wrong on the live website?

Go back to the editor, fix the issue, and publish the changes.
