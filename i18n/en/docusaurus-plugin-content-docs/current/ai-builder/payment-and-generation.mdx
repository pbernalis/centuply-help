# Payment and Generation: what happens after checkout

In Centuply, website generation starts after payment is completed and the basic project details are confirmed.

Payment connects the website draft to your account, so you can edit it and publish it when you are ready.

## When to use this guide

Use this guide when:

- you are at checkout,
- you want to know what happens after payment,
- website generation did not start immediately,
- you see payment pending,
- you see generation in progress,
- you want to know when the editor will open,
- you want to check whether the website was generated correctly.

## Step 1 — Review the summary before payment

Before paying, review the website summary.

Make sure these are correct:

- business type,
- business name,
- language,
- services,
- website goal,
- location,
- core features,
- account email.

If something is wrong, go back and fix it before continuing.

## Step 2 — Continue to checkout

When the summary is correct, continue to checkout.

There you will see payment information, the amount, and details related to the website project.

Do not close the browser tab while payment is in progress.

## Step 3 — Complete payment

Enter payment details and complete the process.

Payment may take a few seconds to confirm.

Possible statuses:

- Processing,
- Pending,
- Paid,
- Failed,
- Cancelled.

If payment appears as paid, Centuply can continue to website generation.

## Step 4 — Wait for confirmation

After payment, wait for the app to return to Centuply.

Usually, you will see a message that payment is being confirmed or that generation is being prepared.

Do not pay again unless you are sure the first attempt failed.

## Step 5 — Generation starts

After payment is confirmed, Centuply starts creating the website.

At this stage, it may generate:

- initial website structure,
- pages,
- sections,
- headlines,
- copy,
- CTAs,
- services,
- basic content organization,
- initial tone.

Generation may take some time depending on the project.

## Step 6 — Avoid unnecessary refresh

During generation, avoid repeated refreshes or going back in the browser.

If the app shows that generation is in progress, wait.

If it seems stuck for a long time, reopen the project from Site Cockpit or check whether there is an error message.

## Step 7 — Open the generated draft

When generation is complete, the website draft will open or you will see an option to open the editor.

The website is not necessarily live yet.

It is an initial draft that you can review, correct, and improve before publishing.

## Step 8 — Review the first result

Before making changes, review the website as a whole.

Check:

- whether the business is described correctly,
- whether pages make sense,
- whether services are correct,
- whether CTAs lead to the right action,
- whether the tone fits,
- whether something important is missing,
- whether any generic phrases need improvement.

## Step 9 — Continue in the editor

After generation, you can edit the website.

You can change:

- text,
- images,
- sections,
- pages,
- buttons,
- forms,
- SEO,
- colors,
- domain,
- publish settings.

The generated draft is a starting point, not a final mandatory result.

## Step 10 — What to do if payment succeeded but you do not see the site

If payment completed but you cannot see the website:

1. Refresh the dashboard.
2. Open Site Cockpit.
3. Check whether there is a draft project.
4. Check payment status in Billing.
5. Check whether there is a generation failed message.
6. Do not pay again before confirming the status.
7. Contact support if the issue continues.

## Step 11 — What to do if payment failed

If payment failed:

- check payment details,
- check whether your bank has a restriction,
- try again later,
- use another payment method if available,
- check whether failed status appears,
- do not start a new payment if the previous one is still processing.

## Step 12 — Before asking for help

If you need support, keep:

- account email,
- project name,
- payment date,
- payment status,
- error screenshot,
- what appears in Site Cockpit,
- whether redirect happened after checkout.

Do not send full card details.

## Best practices

- Review the summary before paying.
- Do not close the tab during payment.
- Do not pay a second time without checking.
- Wait for generation to complete.
- Open the draft and review carefully.
- Make changes in the editor before publishing.
- Check mobile, forms, and CTAs before publishing.

## Frequently asked questions

### Does the website go live immediately after payment?

Not necessarily. Usually, a draft is created first so you can review and edit before publishing.

### What should I do if payment is pending?

Wait a little and check the status again. Do not pay again immediately.

### What if generation takes too long?

Reopen the project from Site Cockpit and check for an error message or generation status.

### Can I change the website after generation?

Yes. You can edit the website inside the editor.

### Should I publish immediately?

No. It is better to check text, images, forms, mobile view, and SEO first.
